## How to read a week

Each week lists what you **build**, the **courses** and **papers** behind it,
anything you **ship**, and the Friday **prove-it**. The tags match the chips in the
[tracker](../tracker/):

| Tag | Meaning |
|-----|---------|
| `BUILD` | write the code / do the work |
| `COURSE` | course material to move through |
| `PAPER` | read it, and be able to defend it |
| `SHIP` | publish something public |
| `PROVE-IT` | Friday: rebuild from memory + a written explainer |
| `GATE` | end-of-month, no-notes checkpoint — don't advance until you pass |

The code you write lands in **[modules/](../modules/)**. Every resource linked across
these files is collected in **[docs/resources.md](../docs/resources.md)**. The
day-to-day rules (schedule, minimum-viable weeks, hackathons) are in
**[docs/how-it-works.md](../docs/how-it-works.md)**, and the compute plan is in
**[docs/compute.md](../docs/compute.md)**.
