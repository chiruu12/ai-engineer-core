# How it works

The operating rules behind the [roadmap](../roadmap/). [← back to the hub](../README.md)

## The schedule

- **~30 hours a week, minimum — Monday to Friday.** Saturday and Sunday are off.
- **Flexible daily timing, fixed weekly quota.** A slow Tuesday is fine if the week still lands at 30h.
- **A day job counts as applied practice, not quota.** The 30h is deliberate learning on top.

## Prove-its

Every **Friday** you rebuild the week's core artifact from a blank file — no
reference — and write a short explainer. If you can't rebuild it, you didn't learn
it; that gap is next week's first priority. The write-up is the public proof the
learning stuck, and it doubles as the seed of a portfolio.

## Gates

Each month ends with a **no-notes gate** — a handful of things you must be able to do
cold (derive, explain, or diagnose). You don't advance to the next month until you
pass. Gates are listed at the bottom of each [month file](../roadmap/).

## From build to project

Scratch code isn't the deliverable — a small, presentable project is. When a build
works and its prove-it passes, promote it (README → clean code → pinned deps → one
test/example → write-up). The full recipe and the running ledger live in
**[modules/README.md](../modules/README.md)**.

## When a week collapses

Life happens — a crunch at work, a hackathon, a bad week. Two rules keep the sprint
honest instead of quietly derailing it:

- **Minimum viable week:** 14 hours of deep work **plus** the Friday prove-it. Courses
  and papers slide to the **buffer week (Week 17)**; the build and the prove-it do not.
- **Hackathon weeks:** if the work is ML-adjacent it counts toward that week's build;
  if it isn't, it burns buffer — it never replaces a prove-it or a gate.

Week 17 is deliberate slack for exactly this. Don't spend it early unless a week
actually collapses.

## Tracking

Progress lives in the [interactive tracker](../tracker/): check items off and it
computes whether you're ahead or behind **from your own start date**. See
[tracker/README.md](../tracker/README.md) for how it works and how to sync progress
across devices.
